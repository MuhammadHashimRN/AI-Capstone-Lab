LangSmith Observability — Dynamic Inventory Reorder Agent
===========================================================

LangSmith Project URL: https://smith.langchain.com/

INTEGRATION STATUS: IMPLEMENTED
LangSmith tracing is integrated into the following files:
  - config.py          : LANGSMITH_API_KEY, LANGSMITH_PROJECT, enable_langsmith()
  - graph.py           : enable_langsmith() called at import (Lab 3 ReAct loop)
  - multi_agent_graph.py : enable_langsmith() called at import (Lab 4 multi-agent)
  - secured_graph.py   : enable_langsmith() called at import (Lab 6 guardrails)
  - approval_logic.py  : enable_langsmith() called at import (Lab 5 HITL)
  - run_eval.py        : Full trace capture with per-run IDs, URLs, and node-level detail
  - app.py             : Streamlit UI displays LangSmith trace status and node-level data

HOW TO ACTIVATE:
  Set these environment variables before running any agent script:

    export LANGSMITH_API_KEY="lsv2_pt_your_api_key_here"
    export LANGCHAIN_PROJECT="inventory-reorder-agent"

  Or set them in your .env / system environment. The enable_langsmith()
  function in config.py automatically sets LANGCHAIN_TRACING_V2=true and
  propagates the key and project name to the LangChain runtime.

  Get a free API key at: https://smith.langchain.com/ (sign up / Settings / API Keys)

WHAT IS TRACED:
  Once enabled, every LangGraph invocation is traced with:
    - Per-node execution times (agent_node, tool_node, guardrail_node)
    - Token usage per LLM call (prompt + completion tokens)
    - Tool call inputs and outputs
    - Full message history at each state transition
    - Run status (success / error)

  The run_eval.py pipeline additionally captures:
    - LangSmith run ID per test case
    - Direct URL to trace in the LangSmith dashboard
    - Node-level latency breakdown via the LangSmith Client API
    - All trace data saved to evaluation_results.json
